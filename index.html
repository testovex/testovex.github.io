Enterprise Security
Built multi-tenant from day one.
Not bolted on after.

Every enterprise evaluator asks the same question: "How do you guarantee Org A can't see Org B's data?" Our answer is architecture, not just policy.

🔒

Tenant Isolation by Architecture

Every org's data is stored and queried in strict isolation. Cross-org data leakage is architecturally prevented — not just policy-restricted.

👥

Role-Based Access Control

Three roles per org: Owner, Admin, and Member. At the project level, a Viewer role provides read-only access — granular control over who can create, edit, run, and manage tests.

🗝️

Locked Credential Variables

API keys, auth tokens, and prod URLs are set at the org level and locked. Test authors can use them in runs but can never read, edit, or override them. Credentials never live in test code.

🚫

No Cross-Org Access

A user in Org A cannot see, access, or interact with anything in Org B — even on the same platform instance.

⏱️

Runtime-Only Variable Scope

Values extracted from the live page (session tokens, dynamic IDs) exist only for the duration of a test run and are never persisted to storage.

🔐

Session-Only Recorder Auth

The browser recorder uses session-only credentials. No API keys are stored in the extension, reducing credential exposure risk at the capture layer.
